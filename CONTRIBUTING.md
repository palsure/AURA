# Contributing to CodeMind AI

Thank you for your interest in contributing to CodeMind AI! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone <your-fork-url>`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit: `git commit -m "Add feature: your feature"`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

Follow the setup instructions in [README.md](README.md).

## Code Style

### Python (Backend)
- Follow PEP 8 style guide
- Use type hints where possible
- Write docstrings for functions and classes
- Maximum line length: 100 characters

### TypeScript/React (Frontend)
- Use TypeScript for type safety
- Follow React best practices
- Use functional components with hooks
- Use Tailwind CSS for styling

## Testing

Before submitting a PR, please:
- Test your changes locally
- Ensure the backend API works correctly
- Verify the frontend displays correctly
- Check for any console errors

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add new feature`
- `fix: Fix bug in code analysis`
- `docs: Update documentation`
- `refactor: Refactor code structure`
- `test: Add tests`

## Pull Request Process

1. Update documentation if needed
2. Add comments to complex code
3. Ensure your code follows the style guide
4. Test thoroughly
5. Update CHANGELOG.md if applicable

## Questions?

Open an issue for questions or discussions.
